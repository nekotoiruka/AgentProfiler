# セキュリティガイドライン

## 秘密情報の管理
- APIキー、トークン、パスワードは絶対にコードにハードコードしない
- `.env` ファイルで管理し、`.gitignore` に含める
- `.env.example` にはプレースホルダーのみ記載
- CI/CD では GitHub Secrets を使用

## 依存関係
- パッケージバージョンはピン留め（`==` or 正確なバージョン指定）
- 定期的に脆弱性スキャンを実行（Trivy, pip-audit, npm audit）
- 不要な依存は削除する

## CI/CD セキュリティスキャン
- **Gitleaks**: コミット内のシークレット検出
- **Trivy**: 依存関係の脆弱性スキャン（CRITICAL/HIGH で失敗）
- **Bandit**: Python コードの静的セキュリティ解析
- **Semgrep**: パターンベースのSAST（全言語対応）
- **pip-audit**: Python 依存の既知脆弱性チェック
- **npm audit**: Node.js 依存の既知脆弱性チェック
- **Dependency Review**: PR時の新規依存の安全性確認

## コーディング時の注意
- SQL: パラメータバインディング必須（文字列結合でクエリを構築しない）
- 入力検証: Pydantic モデルで自動バリデーション
- エラーレスポンス: 内部実装の詳細を露出しない
- CORS: 明示的に許可するオリジンのみ設定
- ファイルパス: ユーザー入力からパスを構築しない（パストラバーサル防止）
- LLM連携: ユーザー入力をプロンプトに含める際はプロンプトインジェクション対策を考慮

## ワークフロー設定
- セキュリティスキャンは `.github/workflows/security.yml` で定義
- PRマージ前に全スキャンをパスすることを推奨
- 週次定期スキャンで新規脆弱性を早期検出
